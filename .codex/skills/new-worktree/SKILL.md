# New Worktree

## 基本方針
- 必ず `gwm` を使って worktree を作成する。
- worktree に移動してから実装・検証・レビューを行う。
- 既存の無関係な変更は触らない。

## 進行フロー
1. 依頼の種別を判定する（実装作業 / PR レビュー）。
2. 対象ブランチまたは PR を特定する。
3. `gwm` で worktree を作成する。
4. worktree 内で作業またはレビューを実施する。

## PR レビュー
- PR番号またはブランチ名が未指定なら、必ず次を確認する。
  - `レビュー対象の PR 番号またはブランチ名を指定してください。`
- 指定を受けたら `gwm` で該当 worktree を作成し、レビューを進める。

## 実装作業
- 依頼内容を実装し、必要なテストや検証を行う。
- 完了時は必ず次を共有する。
  - 変更内容の要約
  - 実行したテストと結果
- 共有後に必ず次を確認する。
  - `この内容で PR を作成しますか？`
